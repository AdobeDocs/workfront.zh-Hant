# 移除預覽反白顯示(Workfront)

## 範圍

僅在&#x200B;**all**&#x200B;為true時套用：

1. 使用者叫用這個工作流程（例如，顯示&#x200B;**「移除預覽反白顯示」**&#x200B;或意圖明顯相同）。
2. Markdown檔案的路徑&#x200B;**不**&#x200B;包含&#x200B;**`product-announcements`** （排除整個資料夾樹狀結構，例如`help/quicksilver/product-announcements/`下的發行說明、Beta和公告）。
3. Markdown檔案是&#x200B;**不是**，列在下方&#x200B;**[排除的路徑](#excluded-paths)**&#x200B;下。
4. Markdown檔案的frontmatter包括&#x200B;**`Courtney`**&#x200B;行上的`author:` （單一作者或共同作者）。
5. 文章有&#x200B;**至少一個**：
   - 預覽環境&#x200B;**內文或真實片段段落中的語言** （典型模式：「醒目提示資訊」、「預覽環境」、「尚未普遍可用」、快速發行說明） — **並非**&#x200B;目錄/索引頁面上&#x200B;**僅連結文字**&#x200B;的相符專案（請參閱下文）；或
   - 任何具有&#x200B;**`class="preview"`**&#x200B;的HTML元素（例如`<span class="preview">`、`<div class="preview">`）；或
   - 預覽程式碼片段&#x200B;**變數**，例如&#x200B;**`{{highlighted-preview}}`**&#x200B;或&#x200B;**`{{highlighted-preview-article-level}}`**。

如果範圍不清楚，請先確認，然後再進行編輯。

**目錄/索引頁面 — 一律略過此案例：** **永不**&#x200B;在&#x200B;**僅**&#x200B;預覽相關文字為&#x200B;**內部** Markdown連結的顯示文字指向&#x200B;**其他**&#x200B;篇文章（例如&#x200B;*在預覽沙箱環境中傳送報告*） **和**，檔案有&#x200B;**no** `class="preview"`、**no**&#x200B;程式碼片段變數，以及&#x200B;**no**&#x200B;在&#x200B;**prose中的預覽樣板（連結外部）**。 這並非在該頁面上預覽反白顯示，而只是目錄提及。 套用至任何索引/目錄，而非只有一個檔案。

### 排除的路徑

請勿將這些專案新增至詳細目錄，或在此工作流程中進行編輯，除非使用者明確覆寫：

- `help/quicksilver/documents/adobe-workfront-for-experience-manager-assets-essentials/send-to-aem.md` — 平行預覽和生產限制需要自動化以外的刻意編輯決定。
- `help/quicksilver/reports-and-dashboards/reports/creating-and-managing-reports/create-manage-reports.md` — 報告TOC；唯一的「預覽」訊號是預覽Sandbox傳遞文章的連結。

## 必要的工作流程（人在回圈中）

請&#x200B;**不**&#x200B;未經核准大量編輯存放庫。

1. **庫存**\
   建立符合上述範圍規則的路徑排序清單（搜尋存放庫；偏好使用`help/`個樹狀結構）。 **省略**&#x200B;底下的任何路徑&#x200B;**`product-announcements`**、**[排除的路徑](#excluded-paths)**&#x200B;底下的任何路徑，以及符合範圍底下&#x200B;**目錄/索引頁面**&#x200B;的任何&#x200B;**目錄/索引**&#x200B;頁面。 如果使用者說列出的檔案沒有預覽反白顯示，請從執行中移除它並收緊條件，而不是強制進行編輯。

2. **開始**\
   詢問是否以清單中的&#x200B;**前**&#x200B;篇文章開頭（或使用者名稱的路徑）。

3. **每篇文章 — 先顯示，然後編輯**
   - 讀取檔案。
   - 明確地提出編輯建議：在節選之前/之後&#x200B;**&#x200B;**&#x200B;或將會變更的重點差異樣式說明。
   - **在寫入檔案之前等待**&#x200B;明確核准（例如：「確定」、「套用」、「是」）。

4. **在每個檔案之後**\
   詢問是否要移至&#x200B;**下一個**&#x200B;文章（或停止/略過）。

## 內容規則（GA：預覽內容會成為檔案）

移除&#x200B;**一般可用性**&#x200B;的預覽反白顯示時，目標為： **保留為預覽**&#x200B;所記錄的行為，卸除&#x200B;**廢棄的「生產中」/舊處理序**&#x200B;分支，然後&#x200B;**移除僅預覽標籤和包裝函式**，讓主題對所有客戶顯示為最新。

### 平行步驟

- 如果文章有一個步驟（或編號專案） **框架為「生產中」** （或同等專案：目前的生產/現有的生產行為） **和** **平行**&#x200B;步驟，在預覽中框架為「**」** （或預覽環境）：
   - **移除**&#x200B;生產中的步驟（舊路徑）。
   - **保留**&#x200B;預覽中步驟的&#x200B;**substance**，但&#x200B;**重新命名**，因此它&#x200B;**不是**&#x200B;預覽專用（移除[預覽中]、[預覽環境]等）。 調整編號，使清單保持一致。

如果結構模稜兩可（沒有明確的平行專案），**停止**&#x200B;並顯示兩個候選者；詢問使用者要保留哪個區塊。

### 平行截面

- 如果&#x200B;**區段** （標題+內文）是&#x200B;**關於「在生產環境中」**，並且有&#x200B;**平行區段** **關於「在預覽環境中」**：
   - **移除**&#x200B;生產環境區段（取代的內容）。
   - **將**&#x200B;取代為預覽區段的內容，然後&#x200B;**移除**&#x200B;預覽環境用語和任何&#x200B;**`class="preview"`**&#x200B;包裝函式，讓區段保持中性（GA就緒）。

### 程式碼片段和文章頂端注意事項

- 功能正式發行後，移除&#x200B;**僅限預覽的樣版**&#x200B;區塊（橫跨/div或僅說明反白內容為僅限預覽、快速發行、沙箱等的段落）。
- 移除其&#x200B;**僅**&#x200B;目的為預覽可用性的連結或句子，除非使用者要求保留不同的通知。

### HTML `class="preview"`

- 針對使用&#x200B;**的元素移除**&#x200B;開頭和結尾標籤&#x200B;**`class="preview"`**，**保留內部內容** （標籤內的markdown/HTML）。
- 處理&#x200B;**`<span class="preview">`**&#x200B;和&#x200B;**`<div class="preview">`**，以及其他標籤上的相同模式（例如&#x200B;**`<p class="preview">`**、**`<li class="preview">`**），當它們出現在&#x200B;**可見** （非註解）內文內容中時。
- 保留清單/表格結構；修正在展開後中斷的清單或零星的空白字元。

### 註解的區段（HTML註解）

- **不要**&#x200B;刪除、取消繞排或以其他方式&#x200B;**修改** **`<!-- … -->`** HTML註解內的任何內容&#x200B;**，除非使用者明確指出該做什麼（例如，刪除整個註解、僅移除註解內的預覽類別、取消註解GA）。**
- 如果預覽相關內容或&#x200B;**`class="preview"`**&#x200B;僅出現在評論中&#x200B;**僅**，請在檢閱文章時&#x200B;**撥打該號碼**：說評論中的內容並&#x200B;**詢問**&#x200B;該做什麼。 **預設：保留備註的區段不變。**

### 不該做什麼

- 請勿在&#x200B;**`product-announcements`**&#x200B;下的路徑上執行此工作流程（版本注意事項和相關資訊）；詳細目錄必須排除它們。
- 請勿清查或編輯&#x200B;**[已排除的路徑](#excluded-paths)**&#x200B;下所列的路徑，除非使用者明確要求納入路徑。
- **不要**&#x200B;自動移除或編輯&#x200B;**註解的** (`<!-- … -->`)區塊；請遵循上述&#x200B;**註解的區段**。
- 當關於此功能可用性模式&#x200B;**不是**&#x200B;時，請勿移除「預覽」（例如，在不相關的內容中以[產品名稱] (·)預覽Sandbox環境&#x200B;**） — 使用判斷並詢問是否不確定。**
- 除非使用者要求，否則請勿變更`author:`或不相關的前置專案。
- 不要略過&#x200B;**顯示→核准**&#x200B;步驟。

## 在展示編輯內容之前的品質檢查

- 議定的變更範圍中沒有剩餘的&#x200B;**`class="preview"`**。
- 沒有孤立的重複標題或中斷的步驟編號。
- 簡介可正確讀取&#x200B;**而非**&#x200B;與GA相抵觸（對於已送出的功能，「僅限預覽」）。
