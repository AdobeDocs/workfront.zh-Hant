# 項目、任務和問題日期概覽，位於 [!DNL Adobe Workfront]

<!--
<p data-mc-conditions="QuicksilverOrClassic.Draft mode">(NOTE: consider expanding on this article with ALL dates for PTIs - Hand off dates, Approval Dates, etc) </p>
-->

本文提供與下列項目、任務和問題相關的最常見日期的定義： [!DNL Adobe Workfront]. 此處包含的影像範例說明日期在Workfront中顯示，且並非詳盡無遺。 還有其他區域會顯示日期。 項目、任務和問題報告和清單中也顯示所有日期。

如需報表和清單的相關資訊，請參閱下列文章：

* [開始使用 [!DNL Adobe Workfront]](../../../workfront-basics/navigate-workfront/use-lists/view-items-in-a-list.md)
* [開始使用 [!DNL Adobe Workfront]](../../../reports-and-dashboards/reports/reporting/get-started-reports-workfront.md)

有關項目、任務和問題欄位的詳細資訊，請參閱 [辭匯表 [!DNL Adobe Workfront] 術語](../../../workfront-basics/navigate-workfront/workfront-navigation/workfront-terminology-glossary.md).

## [!UICONTROL 計畫開始日期]

此 [!UICONTROL 計劃開始日期] 是計劃開始項目、任務或問題的日期。

視 [!UICONTROL 任務約束]，您可能無法編輯 [!UICONTROL 計劃開始日期] 任務。 視 [!UICONTROL 排程模式] ，您可能無法編輯 [!UICONTROL 計劃開始日期] 一個項目。

如需詳細資訊，請參閱 [專案概觀 [!UICONTROL 計劃開始日期]](../../../manage-work/projects/planning-a-project/project-planned-start-date.md).

![](assets/planned-start-date-on-edit-task-highlighted-nwe-350x249.png)

![](assets/planned-start-date-in-task-list-highlighted-nwe-350x167.png)

## [!UICONTROL 計畫完成日期]

此 [!UICONTROL 計畫完成日期] 或 [!UICONTROL 到期日] 日期是計畫完成項目、任務或問題的日期。

視 [!UICONTROL 任務約束]，您可能無法編輯 [!UICONTROL 計畫完成日期] 任務。 視 [!UICONTROL 排程模式] ，您可能無法編輯 [!UICONTROL 計畫完成日期] 一個項目。

此 [!UICONTROL 計畫完成日期] 在 [!DNL Workfront].

如需詳細資訊，請參閱下列文章：

* [任務概述 [!UICONTROL 計畫完成日期]](../../../manage-work/tasks/task-information/task-planned-completion-date.md)
* [設定專案 [!UICONTROL 計畫完成日期]](../../../manage-work/projects/planning-a-project/project-planned-completion-date.md)
* [問題概觀 [!UICONTROL 計畫完成日期]](../../../manage-work/issues/issue-information/issue-planned-completion-date.md)

![](assets/project-header-planned-completion-date-highlighted-nwe-350x34.png)

![](assets/planned-completion-date-in-task-list-highlighted-nwe-350x183.png)

## [!UICONTROL 輸入日期]

此 [!UICONTROL 參加日期] 是在Workfront中建立專案、任務或問題的日期。

此 [!UICONTROL 參加日期] 不會影響專案、工作或問題的時間軸，但對於追蹤和報告用途而言，這很重要。 [!DNL Workfront] 自動產生 [!UICONTROL 參加日期] 對象建立時，您無法手動編輯它。

![](assets/entry-date-in-task-details-highlighted-nwe-350x105.png)

## [!UICONTROL 實際開始日期]

此 [!UICONTROL 實際開始日期] 是使用者實際開始處理專案、任務或問題的日期。 此 [!UICONTROL 實際開始日期] 在建立專案、任務或問題時為空。

您可以手動指出工作何時開始於任務或問題，或 [!UICONTROL 實際開始日期] 當任務或問題狀態從 [!UICONTROL 新增] to [!UICONTROL 進行中] 或 [!UICONTROL 完成]. 此 [!UICONTROL 實際開始日期] 的日期與項目上第一個任務開始的日期相同。

>[!TIP]
>
>此 [!UICONTROL 實際開始日期] 可能不符合 [!UICONTROL 計劃開始日期] 項目、任務或問題，因為用戶可能在計畫日期之前或之前開始工作。

如需詳細資訊，請參閱 [專案概觀 [!UICONTROL 實際開始日期]](../../../manage-work/projects/planning-a-project/project-actual-start-date.md).

>[!NOTE]
>
>此 [!UICONTROL 必須開始] 任務或「固定日期」約束會影響 [!UICONTROL 計劃開始日期] 任務，而不是 [!UICONTROL 實際開始日期]. 這會更新 [!UICONTROL 計劃開始日期] 至您指定的日期。 此 [!UICONTROL 實際開始日期] 會獨立於 [!UICONTROL 計劃開始日期]，如上所述。

![](assets/actual-start-date-on-edit-task-highlighted-nwe-350x251.png)

## [!UICONTROL 實際完成日期]

此 [!UICONTROL 實際完成日期] 是使用者實際完成專案、任務或問題的日期。 此 [!UICONTROL 實際完成日期] 在建立專案、任務或問題時為空。

您可以手動指出工作何時完成任務或問題，或 [!UICONTROL 實際完成日期] 會在發生下列任一情況時自動填入：

* 項目、任務或問題狀態更改為 [!UICONTROL 完成], [!UICONTROL 已關閉]，或 [!UICONTROL 已解決].
* 任務或項目完成百分比為100%。

此 [!UICONTROL 實際完成日期] 的日期與項目上第一個任務完成的日期相同。

>[!TIP]
>
>此 [!UICONTROL 實際完成日期] 不符合 [!UICONTROL 計畫完成日期].

如需詳細資訊，請參閱 [專案概觀 [!UICONTROL 實際完成日期]](../../../manage-work/projects/planning-a-project/project-actual-completion-date.md).

![](assets/actual-completion-date-task-details-highlighted-nwe-350x189.png)

## [!UICONTROL 認可日期]

此 [!UICONTROL 提交日期] 是指派給任務或問題的用戶提交完成任務或問題的日期。 這與 [!UICONTROL 計畫完成日期]，因為這是僅由負責工作的使用者提供的完成日期更實際的估計。 如需詳細資訊，請參閱 [[!UICONTROL 提交日期] 概述](../../../manage-work/projects/updating-work-in-a-project/overview-of-commit-dates.md).

>[!NOTE]
>
>變更 [!UICONTROL 提交日期] 會影響 [!UICONTROL 預計完成日期] 但不是 [!UICONTROL 計畫完成日期] 任務或問題。 項目經理可以使用受託人對 [!UICONTROL 提交日期] 更新 [!UICONTROL 計畫完成日期] 任務或問題。

![](assets/commit-date-on-task-details-highlighted-nwe-350x193.png)

![](assets/commit-date-in-updates-stream-on-task-nwe-350x344.png)

## [!UICONTROL 預計開始日期]

此 [!UICONTROL 預計開始日期] 是項目、任務或問題的即時日期，並考慮所有延遲。 這是比 [!UICONTROL 計劃開始日期]. 此 [!UICONTROL 計劃開始日期] 不考慮延遲或過去日期。

當您首次規劃專案時， [!UICONTROL 計劃開始日期] 和 [!UICONTROL 預計開始日期] 任務和項目的任務是相同的。 由於延遲可能發生，或任務可能提前完成，因此 [!UICONTROL 預計開始日期] 會變得與 [!UICONTROL 計劃開始日期].

對於任務， [!UICONTROL 預計開始日期] 也可能與 [!UICONTROL 計劃開始日期] 當其中一位前任落後時。

>[!TIP]
>
>您可以檢視 [!UICONTROL 預計開始日期] 問題。

如需詳細資訊，請參閱 [專案概觀 [!UICONTROL 預計開始日期]](../../../manage-work/projects/planning-a-project/project-projected-start-date.md).

![](assets/projected-start-date-in-task-details-highlighted-nwe-350x188.png)

## P[!UICONTROL 預計完成日期]

此 [!UICONTROL 預計完成日期] 是即時計算的指標，指出項目、任務或問題何時完成。 將項目、任務或問題標籤為已完成時， [!UICONTROL 預計完成日期] 日期變更 [!UICONTROL 實際完成日期].

如果一切順利如計畫 [!UICONTROL 預計完成日期] 應符合 [!UICONTROL 計畫完成日期]. 否則，由於前身任務的延遲， [!UICONTROL 預計完成日期] 可能會與 [!UICONTROL 計畫完成日期].

如需詳細資訊，請參閱 [概觀 [!UICONTROL 預計完成日期] 針對專案、工作和問題](../../../manage-work/projects/planning-a-project/project-projected-completion-date.md).

![](assets/projected-completion-date-in-task-details-highlighted-nwe-350x187.png)

## [!UICONTROL 小時參加日期]

當您記錄項目、任務和問題的時間，以指示您在項目、任務或問題上實際花費的時間（以小時計）時，您的日誌時間將變成 [!UICONTROL 實際小時數] 項目、任務或問題。

您記錄時間的日期為 [!UICONTROL 小時參加日期] 」欄位。

>[!TIP]
>
>此 [!UICONTROL 小時參加日期] 與 [!UICONTROL 參加日期] 這不是建立小時日誌的日期，而是您要關聯小時的日期。

您可以在 [!UICONTROL 專案], [!UICONTROL 任務]，或 [!UICONTROL 問題更新] 區段或 [!UICONTROL 小時] 區段。 在 [!UICONTROL 小時] 區段中，您可以手動指定小時登入日期及小時所屬的使用者。

如需詳細資訊，請參閱 [記錄時間](../../../timesheets/create-and-manage-timesheets/log-time.md).

>[!TIP]
>
>我們建議您將時間記錄在工作任務和問題上，而非父任務或專案。 工作任務記錄的時間累計至父任務和項目，如 [!UICONTROL 實際小時數] ，以執行父任務和專案。 登入時間問題累計至專案為 [!UICONTROL 實際小時數] 來取得。

![](assets/log-time-box-task-hours-section-nwe-350x500.png)

![](assets/log-time-in-update-stream-task-nwe-350x185.png)

您可以檢視 [!UICONTROL 小時參加日期] 小時報告和清單。
