# 中的專案、任務和問題日期概觀 [!DNL Adobe Workfront]

<!--
<p data-mc-conditions="QuicksilverOrClassic.Draft mode">(NOTE: consider expanding on this article with ALL dates for PTIs - Hand off dates, Approval Dates, etc) </p>
-->

本文提供中與專案、任務和問題相關的最常見日期的定義。 [!DNL Adobe Workfront]. 此處包含的影像範例說明日期在Workfront中顯示，並非完整日期。 還有其他區域會顯示日期。 所有日期也會顯示在專案、任務和問題報告與清單中。

如需關於報告和清單的資訊，請參閱下列文章：

* [開始使用中的清單 [!DNL Adobe Workfront]](../../../workfront-basics/navigate-workfront/use-lists/view-items-in-a-list.md)
* [開始使用報告](../../../reports-and-dashboards/reports/reporting/get-started-reports-workfront.md)

如需專案、任務和問題欄位的詳細資訊，請參閱 [字彙表 [!DNL Adobe Workfront] 術語](../../../workfront-basics/navigate-workfront/workfront-navigation/workfront-terminology-glossary.md).

## [!UICONTROL 計劃開始日期]

此 [!UICONTROL 計劃開始日期] 是專案、任務或問題計劃開始的日期。

依據 [!UICONTROL 任務限制]，您可能無法編輯 [!UICONTROL 計劃開始日期] 任務的。 依據 [!UICONTROL 排程模式] 您可能無法編輯此專案的 [!UICONTROL 計劃開始日期] 專案的。

如需詳細資訊，請參閱 [專案概述 [!UICONTROL 計劃開始日期]](../../../manage-work/projects/planning-a-project/project-planned-start-date.md).

![](assets/planned-start-date-on-edit-task-highlighted-nwe-350x249.png)

![](assets/planned-start-date-in-task-list-highlighted-nwe-350x167.png)

## [!UICONTROL 計畫完成日期]

此 [!UICONTROL 計畫完成日期] 或 [!UICONTROL 到期日為] 日期是專案、任務或問題計畫完成的日期。

依據 [!UICONTROL 任務限制]，您可能無法編輯 [!UICONTROL 計畫完成日期] 任務的。 依據 [!UICONTROL 排程模式] 您可能無法編輯此專案的 [!UICONTROL 計畫完成日期] 專案的。

此 [!UICONTROL 計畫完成日期] 在某些區域顯示為到期日期 [!DNL Workfront].

如需詳細資訊，請參閱下列文章：

* [任務概觀 [!UICONTROL 計畫完成日期]](../../../manage-work/tasks/task-information/task-planned-completion-date.md)
* [設定專案 [!UICONTROL 計畫完成日期]](../../../manage-work/projects/planning-a-project/project-planned-completion-date.md)
* [問題概觀 [!UICONTROL 計畫完成日期]](../../../manage-work/issues/issue-information/issue-planned-completion-date.md)

![](assets/project-header-planned-completion-date-highlighted-nwe-350x34.png)

![](assets/planned-completion-date-in-task-list-highlighted-nwe-350x183.png)

## [!UICONTROL 輸入日期]

此 [!UICONTROL 輸入日期] 是在Workfront中建立專案、任務或問題的日期。

此 [!UICONTROL 輸入日期] 不影響專案、任務或問題的時間表，但用於追蹤和報告用途很重要。 [!DNL Workfront] 自動產生 [!UICONTROL 輸入日期] 物件建立後，您無法手動編輯它。

![](assets/entry-date-in-task-details-highlighted-nwe-350x105.png)

## [!UICONTROL 實際開始日期]

此 [!UICONTROL 實際開始日期] 是使用者實際開始處理專案、任務或問題的日期。 此 [!UICONTROL 實際開始日期] 在建立專案、任務或問題時為空白。

您可以手動指出工作何時開始進行一項任務或一個問題，或 [!UICONTROL 實際開始日期] 當任務或問題狀態從變更時自動填入 [!UICONTROL 新增] 至 [!UICONTROL 進行中] 或 [!UICONTROL 完成]. 此 [!UICONTROL 實際開始日期] 的日期與專案上第一個任務的開始日期一致。

>[!TIP]
>
>此 [!UICONTROL 實際開始日期] 可能不符合 [!UICONTROL 計劃開始日期] 專案、任務或問題的日期，因為使用者可能在其計畫日期之前或之後開始工作。

如需詳細資訊，請參閱 [專案概述 [!UICONTROL 實際開始日期]](../../../manage-work/projects/planning-a-project/project-actual-start-date.md).

>[!NOTE]
>
>此 [!UICONTROL 必須開始於] 任務或固定日期限制會影響 [!UICONTROL 計劃開始日期] 而不是任務的 [!UICONTROL 實際開始日期]. 這會更新 [!UICONTROL 計劃開始日期] 到您指定的日期。 此 [!UICONTROL 實際開始日期] 獨立更新 [!UICONTROL 計劃開始日期]，如上所述。

## [!UICONTROL 實際完成日期]

此 [!UICONTROL 實際完成日期] 是使用者實際完成專案、任務或問題的日期。 此 [!UICONTROL 實際完成日期] 在建立專案、任務或問題時為空白。

您可以手動指出任務或問題的工作完成時間，或 [!UICONTROL 實際完成日期] 發生下列任一情況時自動填入：

* 專案、任務或問題狀態變更為 [!UICONTROL 完成]， [!UICONTROL 已關閉]，或 [!UICONTROL 已解決].
* 任務或專案完成百分比為100%。

此 [!UICONTROL 實際完成日期] 的日期與專案上第一個任務完成的日期一致。

>[!TIP]
>
>此 [!UICONTROL 實際完成日期] 可能不符合 [!UICONTROL 計畫完成日期].

如需詳細資訊，請參閱 [專案概述 [!UICONTROL 實際完成日期]](../../../manage-work/projects/planning-a-project/project-actual-completion-date.md).

![](assets/actual-completion-date-task-details-highlighted-nwe-350x189.png)

## [!UICONTROL 認可日期]

此 [!UICONTROL 認可日期] 是使用者指派到任務或問題認可完成任務或問題的日期。 這與 [!UICONTROL 計畫完成日期]，因為這是僅由負責工作的使用者提供的對完成日期的更實際估計。 如需詳細資訊，請參閱 [[!UICONTROL 認可日期] 概述](../../../manage-work/projects/updating-work-in-a-project/overview-of-commit-dates.md).

>[!NOTE]
>
>變更 [!UICONTROL 認可日期] 影響 [!UICONTROL 預計完成日期] 但不是 [!UICONTROL 計畫完成日期] 任務或問題的摘要。 專案經理可使用受指派人對 [!UICONTROL 認可日期] 更新 [!UICONTROL 計畫完成日期] 任務或問題的摘要。

![](assets/commit-date-on-task-details-highlighted-nwe-350x193.png)

![](assets/commit-date-in-updates-stream-on-task-nwe-350x344.png)

## [!UICONTROL 預計開始日期]

此 [!UICONTROL 預計開始日期] 是專案、任務或問題的即時日期，會考慮所有延遲。 此專案、任務或問題的開始日期比 [!UICONTROL 計劃開始日期]. 此 [!UICONTROL 計劃開始日期] 未將延遲或過去的日期納入考量。

當您第一次計畫專案時， [!UICONTROL 計劃開始日期] 和 [!UICONTROL 預計開始日期] 的相同專案。 由於可能會發生延遲或工作可能會提前完成，因此 [!UICONTROL 預計開始日期] 可能會變得與 [!UICONTROL 計劃開始日期].

若為工作，則為工作 [!UICONTROL 預計開始日期] 也可能與其 [!UICONTROL 計劃開始日期] 當其中一個前置任務落後於排程時。

>[!TIP]
>
>您可以檢視 [!UICONTROL 預計開始日期] 問題僅存在於清單或報告中。

如需詳細資訊，請參閱 [專案概述 [!UICONTROL 預計開始日期]](../../../manage-work/projects/planning-a-project/project-projected-start-date.md).

![](assets/projected-start-date-in-task-details-highlighted-nwe-350x188.png)

## P[!UICONTROL 專案完成日期]

此 [!UICONTROL 預計完成日期] 是專案、任務或問題將完成的即時計算量度指標。 將專案、任務或問題標示為「已完成」時， [!UICONTROL 預計完成日期] 對日期所做的變更 [!UICONTROL 實際完成日期].

如果一切順利，並如計畫進行 [!UICONTROL 預計完成日期] 應該符合 [!UICONTROL 計畫完成日期]. 否則，由於前置任務上的延遲， [!UICONTROL 預計完成日期] 可能會與 [!UICONTROL 計畫完成日期].

如需詳細資訊，請參閱 [概述 [!UICONTROL 預計完成日期] 針對專案、任務和問題](../../../manage-work/projects/planning-a-project/project-projected-completion-date.md).

![](assets/projected-completion-date-in-task-details-highlighted-nwe-350x187.png)

## [!UICONTROL 小時輸入日期]

當您為專案、任務和問題記錄時間，以指出您在專案、任務或問題上花費的實際時間（以小時為單位）時，您記錄的時間會變成 [!UICONTROL 實際小時] 專案、任務或問題的許可權。

您記錄時間的日期是 [!UICONTROL 小時輸入日期] 時數專案上的欄位。

>[!TIP]
>
>此 [!UICONTROL 小時輸入日期] 不同於 [!UICONTROL 輸入日期] 因為這不是建立時數記錄的日期，而是您希望與時數產生關聯的日期。

您可以登入 [!UICONTROL 專案]， [!UICONTROL 任務]，或 [!UICONTROL 問題更新] 區段或 [!UICONTROL 小時] 區段。 在中記錄時間時 [!UICONTROL 小時] 區段，您可以手動指定小時輸入日期以及小時所屬的使用者。

如需詳細資訊，請參閱 [記錄時間](../../../timesheets/create-and-manage-timesheets/log-time.md).

>[!TIP]
>
>我們建議在工作任務和問題上記錄時間，而不是父級任務或專案。 工作任務的登入時間會向上彙整至父任務和專案，如下所示 [!UICONTROL 實際小時] 用於父任務和專案。 登入問題累積至專案的時間為 [!UICONTROL 實際小時] 專案的。

![](assets/log-time-box-task-hours-section-nwe-350x500.png)

![](assets/log-time-in-update-stream-task-nwe-350x185.png)

您可以檢視 [!UICONTROL 小時輸入日期] 在時數報表和清單中。

![](assets/hour-entry-date-in-view-nwe-350x173.png)
