# 組和審批流程更改如何影響分配的審批流程

本文說明當核准程式已與任務、問題、專案、範本或範本任務建立關聯，而Workfront管理員（或具有核准程式管理存取權的使用者）執行下列其中一項作業時，會發生什麼情況：

* 將審批流程（組級）從一個組更改為另一個組
* 更改與項目關聯的組
* 將審批流程從組級更改為系統級
* 將審批流程從系統級更改為組級

本文也說明在不同群組的兩個專案之間移動或複製與群組層級核准程式相關的工作或問題時，會發生什麼事。

如需可在Workfront中使用的3種核准程式類型的相關資訊，請參閱 [核准流程概述](../../../review-and-approve-work/manage-approvals/approval-process-in-workfront.md).

>[!NOTE]
>
>如果項目或其任務或問題已與使用組層自定義狀態的組層審批流程關聯，則更改組可能會在上一組的審批狀態和系統層的現有狀態之間產生衝突。
>
>例如，如果審批流程包含兩個路徑，一個是系統級狀態，另一個是組級狀態，相當於同一系統級狀態，則更改原始項目組將導致Workfront無法了解可能不存在於第二組的組特定狀態。 在這種情況下，您會遇到錯誤。
>
>在更新項目組之前，請考慮刪除項目的組級審批流程，或刪除其任務或問題。
>
>如需建立群組層級核准程式的相關資訊，請參閱 [組級審批流程](../../../administration-and-setup/manage-groups/work-with-group-objects/create-and-modify-groups-approval-processes.md).
>
>如需建立群組層級自訂狀態的相關資訊，請參閱 [建立或編輯群組狀態](../../../administration-and-setup/manage-groups/manage-group-statuses/create-or-edit-a-group-status.md)

## 將任務或問題移動或複製到具有與審批流程不同的組的項目

將任務或問題從項目移動或複製到另一個項目可能會根據這兩個項目的組影響任務或問題的現有審批流程。 下表說明了可能存在的方案：

| 原始任務或發放審批流程 | 兩個項目的組 | 任務或問題移至其他項目後，審批流程中的更改 |
|---|---|---|
| 與全系統狀態關聯的一次性批准過程 | 專案位於相同或不同的群組中 | 無更改 |
| 與組特定狀態關聯的一次性審批流程 | 專案位於不同群組 | 核准仍為一次性核准程式，且與核准更新相關聯的狀態會與新群組的類似狀態相符。 |
| 全系統的批准過程 | 專案位於相同或不同的群組中 | 無更改 |
| 特定於組的審批流程 | 項目在同一組 | 無更改 |
| 特定於組的審批流程 | 專案位於不同的群組，且群組具有不同的群組特定狀態 | 核准會變成一次性核准程式，以及與核准更新相關聯的狀態會與新群組的類似狀態相符。 |
| 特定於組的審批流程 | 項目位於不同的組中，並且新組中的狀態與第一個組的審批流程關聯的狀態具有相同的鍵 | 無更改 |
