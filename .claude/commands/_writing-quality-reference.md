# 寫入品質參考

`writing-quality`技能的詳細規則。 SKILL.md檔案涵蓋最常見的規則；此檔案提供標點符號、術語、大寫範例和程式格式的完整參考。

---

## 標點符號

### 週期

- 如果專案完成一個句子，請在清單中使用句號（「從X功能表，您可以：」）
- 如果任何專案是完整的句子，請在清單中使用句號
- 句點不得使用主要內容格式化(正確：按一下&#x200B;**完成**。 /不正確：按一下&#x200B;**完成。**)
- 在副檔名(.pdf)中使用句號

### 逗號

- 牛津逗號一律：「動機、手段和機會」
- 簡介短語後的逗號：「若要開啟檔案，請按一下[開啟]。」
- 在日期中句子中的年份後面以逗號標示：「The January 2， 2016， issue of...」
- 程式中的「then」前須有逗號：「按一下&#x200B;**安裝程式**，然後按一下&#x200B;**介面**」。
- 無逗號，僅限月年：「2016年1月」

### 單引號

- 用於收縮：不，他們是，它
- 單一佔有式：新增「s （Jen的電腦）」
- 以s為結尾的複數佔有慾清單：僅新增「 （The Joneses」電腦）
- 區分「它」（它是）和「其」（屬於它）

### 冒號

- 冒號後面小寫，除非後面有直接引號、專有名詞或專案符號清單
- 在引入清單的句子/片段結尾使用
- 程式標題後面沒有冒號
- 沒有冒號可引入視覺元素或程式碼區段

### 分號

- 在不結合的情況下連線兩個子句
- 包含其標點符號的個別片語
- 在程式中，請勿在粗體文字後面加上分號
- 不要包含在超連結中

### 引號

- 將其稱為「引號」（而不是「引號」）
- 右引號位於逗號與句點之外
- 問號的位置取決於是引號還是句子是問題

### 連字型大小

- 儘可能避免使用需要連字型大小的結構
- 在名詞前加上連字複合修飾詞：「唯讀記憶體」
- 當一個修飾元為分式時斷字：「有複製保護的磁碟」
- 當一個修飾元為數字時斷字：「12點字型」
- 索引鍵組合中沒有連字型大小 — 使用+ (Ctrl+I)

### 長破折號

請謹慎使用。 請考慮改成重新調整句子。

### 短破折號

- 範圍：2005-2013
- 負號或負數
- 將複合形容詞連線到另一個單字：「dialog box-type options」

### 橢圓

- 省略程式碼或語法
- UI指示將會出現對話方塊
- 省略引號中的文字（如果取代句尾則新增句號）

### 斜線

- 沒有斜線可供選擇 — 使用「或」（「他/她」而不是「他/她」）
- 用於組合：開啟/關閉開關
- 用於網際網路位址和分數

## 大寫範例

### Workfront條款 — 直接與間接參考

| 術語 | 直接（大寫） | 間接（小寫） |
|------|---------------------|----------------------|
| 規劃完成日期 | 「完成&#x200B;**規劃完成日期**&#x200B;欄位。」 | 「每項任務必須有計畫完成日期。」 |
| 版面配置範本 | 「在設定中，移至&#x200B;**配置範本**。」 | 「指派配置範本給每位使用者。」 |
| 群組管理員 | 「找到&#x200B;**群組管理員**&#x200B;欄位。」 | 「請聯絡群組管理員。」 |
| 自訂表單 | 「將&#x200B;**自訂表單**&#x200B;附加至專案。」 | 「您可以建立任何物件的自訂表單。」 |

### 常見Workfront辭彙（參考UI時大寫）

- 自訂表單
- 期間
- 規劃時數
- 規劃完成日期
- 時數類型
- 群組管理員/系統管理員
- Workfront管理員（永遠小寫的「管理員」人員）
- 提醒通知
- 版面配置範本
- 篩選、檢視、分組（在篩選、檢視和分組內容中）
- 資源規劃工具（計畫）

### 按鈕

- 遵循UI大小寫
- 如果UI中的按鈕是全大寫的，請改用句子大小寫寫

## 術語

### UI元素

| 元素 | 正確詞語 |
|---------|-------------|
| 可摺疊的選項清單 | 功能表（新增「下拉式清單」，視需要提供詳細資訊） |
| 無法使用的命令 | &quot;unavailable&quot; （絕不&quot;dimmed&quot;、&quot;disabled&quot;或&quot;grayed&quot;） |
| 包含表單欄位的快顯視窗 | 對話方塊 |
| 未命名的按鈕 | 使用按鈕的工具提示名稱 |
| 錯誤訊息/快顯通知 | 「訊息」（「顯示通知您的訊息……」） |
| 文字輸入 | 欄位 |

### 角色

| 內容 | 術語 |
|---------|------|
| Workfront中的角色 | 系統管理員 |
| 管理Workfront的人員 | Workfront管理員 |

### 覆寫與覆寫

這些是具有不同含義的不同字詞 — 使用正確字詞作為前後關聯。

### 副檔名

請一律使用副檔名，而非縮寫：「上傳.pdf檔案」（而非「上傳PDF」）。

## 程式格式

### 簡介

僅在必要時包含。 偏好的格式（依序）：

1. 不定式字詞+冒號：「若要在您的系統中建立使用者的暫時密碼：」
2. 完整的句子+句號：「您必須為系統中的使用者建立臨時密碼。」
3. 不定式片語+「請遵循這些步驟」+冒號：「若要為使用者建立暫時密碼，請遵循這些步驟：」

### 步驟慣例

- 對所有程式使用編號清單，甚至是單一步驟程式
- 粗體可點按動作和步驟中的UI元素
- 使用特定動作動詞：&quot;type&quot;、&quot;select&quot;、&quot;click&quot; — 避免&quot;set&quot;或&quot;specify&quot;
- 說明系統回應在發生之前或緊接發生之後

系統回應放置的範例：

```
1. Click **Sharing**.
2. In the **Task Access** dialog box, enter the name of the user you want to give task access to.
```

### 條件式與選擇性步驟

```
3. (Conditional) If you are a member of more than one team, select...
3. (Optional) To add an emoji, click...
3. (Conditional and optional) If you want to add a note...
```

### 子步驟和欄位清單

以命令式命令匯入（設定、指定、執行），然後列出欄位：

```
1. Specify the following:
   a. **Favorite color**: Select your favorite color from the drop-down menu.
   b. **Favorite song**: Type your favorite song.
```

### 多種方法

先以最直接的方式記錄：

```
1. In the **Main Menu**, click **Home**.

   Or

   Click the **Home** icon in the upper-left corner.
```

### 直角括弧

- 顯示功能表/索引標籤導覽： 「按一下&#x200B;**電子郵件** > **通知**」
- 粗體按鈕名稱，而非括弧
- 括弧內的空格
- 請勿用於主要功能表 — 使用主要功能表片段

## 粗體和斜體參考

### 粗體 — 用於：

| 內容 | 步驟 | 外部步驟 |
|---------|----------|---------------|
| 可點按的按鈕/連結 | 粗體 | 不要粗體 |
| 功能表名稱 | 粗體 | 不要粗體 |
| 圖示名稱 | 粗體 | 不要粗體 |
| 對話方塊名稱 | 粗體 | 不要粗體 |
| 頁面名稱 | 粗體 | 不要粗體 |
| 欄位清單中的欄位名稱 | 粗體+冒號 | 不要粗體 |

### 斜體 — 用於：

- 文字使用者應輸入：「在URL方塊中輸入&#x200B;*my.workfront.com*」
- 絕不強調重點 — 改為重新建構句子

## 附註、提示、警告 — 詳細指引

### 何時使用每個

**附註** (`>[!NOTE]`)：
- 不符合段落內容的資訊
- 避免使用者感到挫折或浪費時間（「您必須擁有團隊授權才能完成此工作。」）
- 版本設定或相容性資訊（「此功能僅在舊版校訂檢視器中可用。」）

**提示** (`>[!TIP]`)：
- 可讓使用者生活更輕鬆的建議，但並非至關重要
- 範例：「按住Shift鍵可同時共用多個檔案。」

**重要** (`>[!IMPORTANT]`)：
- 步驟或程式的重要資訊

**警告** (`>[!WARNING]`)：
- 提醒使用者可能發生資料遺失

### 規則

- 每個區段最多一個
- 永不棧疊多個附註/提示/警告
- 以專案符號清單將相關附註合併成附註
- 不要使用附註來宣告新功能 — 重寫文章文字

## 互動參照陣列

| 狀況 | 模式 |
|-----------|---------|
| 提供資訊後 | 「如需詳細資訊，請參閱[文章名稱](link)。」 |
| 無先前資訊 | 「如需詳細資訊，請參閱[文章名稱](link)。」 |
| 至另一篇文章中的章節 | 「如需詳細資訊，請參閱[文章名稱](link)中的[區段]。」 |
| 至同一篇文章中的章節 | 「如需詳細資訊，請參閱本文中的[章節]。」 |

刊登：
- 在段落中：內嵌
- 在程式步驟中：步驟後的新行
- 多個引用：專案符號清單

## DQTI品質特性

### 使用簡單

| 特性 | 重要問題 |
|---------------|---------------|
| 任務方向 | 這是否著重於使用者能做什麼？ 標題是否為命令式動詞？ |
| 準確度 | 術語是否與UI完全相符？ 技術細節正確嗎？ |
| 完整性 | 是否有足夠的資訊可執行？ 是否有不必要的資訊要移除？ |

### 易於理解

| 特性 | 重要問題 |
|---------------|---------------|
| 清晰度 | 每一個句子都是一個想法嗎？ 文字是否簡單明瞭？ |
| 具體 | 是否有特定範例？ UI元素的名稱精確嗎？ |
| 樣式 | 主動語態？ 現在時態？ 術語是否一致？ |

### 容易找到

| 特性 | 重要問題 |
|---------------|---------------|
| 組織 | 邏輯結構？ 漸進式公開？ 適用時為最新優先？ |
| 可擷取性 | 讀者可以掃描標題並瞭解內容嗎？ 是否提供連結？ |
| 視覺效果 | 超過3個專案的清單？ 要比較的資料表？ 沒有文字牆段落？ |

## 常見修正

| 問題 | 修正 |
|-------|-----|
| 「我們很高興為您介紹……」 | 移除行銷語言 — 直接說明功能 |
| 「X的設定已更新」 | 「您現在可以設定X」（主動、工作導向） |
| 「建議……」 | 「我們建議……」 |
| 「使用者可利用此功能來……」 | 「您可以使用此功能……」 |
| &quot;set the value&quot; / &quot;specify the option&quot; | &quot;輸入值&quot; / &quot;選取選項&quot; |
| &quot;disabled&quot; / &quot;grayed out&quot; | &quot;unavailable&quot; |
| &quot;dropdown&quot; （獨立） | 「功能表」或「下拉式功能表」 |
| &quot;he/she&quot; | 「他們」或重組 |
| 棧疊備註 | 用專案符號合併成一個附註 |
| 粗體外部程式步驟 | 移除粗體 |
